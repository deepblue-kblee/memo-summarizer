# 🎯 PARA 메모 자동화 전략 및 비전 (Harness Engineering)

이 문서는 `memo-summarizer` 프로젝트의 핵심 설계 원칙과 전략적 지향점을 정의합니다. 모든 에이전트는 이 설계 원칙을 준수하여 작업해야 합니다.

## 🏗️ 핵심 설계 원칙

1.  **정확한 상태 기반 관리 (PARA State Management)**
    *   단순 요약을 넘어, 전체 프로젝트(Projects)와 영역(Areas)의 실시간 상태를 추적합니다.
    *   메모를 통해 할 일의 완료 상태(`- [x]`)를 자동으로 갱신하고, 진척도를 시각화합니다.

2.  **데이터 무결성 및 지능형 병합**
    *   AI 기반 병합 과정에서 기존 데이터(히스토리, 할 일)의 손실을 방지합니다.
    *   중복을 제거하고 가장 최신 정보를 상단에 배치하는 구조를 유지합니다.

3.  **에이전트 가독성 최적화 (Agent Legibility)**
    *   에이전트가 문서를 읽었을 때 현재 상태와 다음에 할 일을 즉각 파악할 수 있도록 구조를 단순화합니다.
    *   인지 부하를 줄이기 위해 파일 당 정보 밀도를 최적으로 관리합니다.

## 🚧 기술적 제약 및 전략

1.  **안티 파편화 (Anti-Sprawl Strategy)**: 새로운 주제는 무분별하게 파일을 생성하지 않고, 기존의 관련 Projects/Areas 파일로 최대한 수렴(Merge)시킵니다.
2.  **비용 최적화 (Cost Efficiency)**: 파일이 커짐에 따라 전체 내용을 AI에게 보내지 않고, 최근 맥락과 할 일 목록만 발췌(Sliding Window)하여 전달하는 구조를 지향합니다.
3.  **지능형 계층화**: 이력이 길어지면 자동 아카이빙을 통해 본문의 크기를 유지하고, 에이전트가 필요할 때만 아카이브를 참조하게 합니다.
