### Phase 2: 가속성 및 비용 최적화 (진행 예정)
- [ ] **메모 분석 호출 최적화 (Batch 처리 도입)**: 개별 파일당 1회 호출 방식에서 여러 파일을 묶어 한 번에 호출하는 방식으로 개선하여 API 호출 횟수 및 비용 절감
- [ ] **완료된 할 일 자동 섹션 이동**: AI 프롬프트 강화하여 완료된 작업을 별도 섹션으로 분리
- [ ] **컨텍스트 발췌 병합**: 파일이 커질 경우 최근 맥락과 할 일만 발췌하여 토큰 절약
- [ ] 메모 분석 결과 캐싱으로 반복 처리 최적화
- [ ] 대량 메모 배치 처리시 병렬 분석 지원

### Phase 3: 지능형 아카이빙 및 검색 (장기 과제)
- [ ] 오래된 이력 자동 아카이빙 기능 (`{topic}_Archive.md`)
- [ ] 아카이브 색인 생성 및 필요 시 참조(Retrieval) 기능
- [ ] 웹 브라우저 기반 시각화 및 통계 대시보드

### Phase 4: Agentic Workflow & Scaling (팀/조직 단위 확장)
- [ ] **Harness Engineering 템플릿화/스킬화**: 본 프로젝트에서 정립한 AI 에이전트 개발 철학과 시스템 구조(로깅, 에러 핸들링, 진입점 분리 등)를 다른 프로젝트에서 즉각 사용할 수 있도록 레포지토리 템플릿 또는 Gemini CLI 스킬로 패키징
- [ ] **CI/CD 파이프라인 통합 (PR 리뷰 봇)**: PR 생성 시 AI가 코드 리뷰 수행. 단순 문법 검사가 아닌 'Harness Engineering' 준수 여부(테스트 누락, 로깅 처리 등)를 중점적으로 검토하도록 프롬프트 엔지니어링 적용 (초기엔 코멘트 전용)
- [ ] **Git Issue 자율 트리아지(Triage) 시스템**: Cron 또는 웹훅 기반으로 새로 생성된 Issue를 AI가 분석하여 라벨링, 초기 원인 분석, 관련 파일 탐색 결과를 코멘트로 자동 작성 (토큰/비용 제어 및 스팸 방지 안전장치 필수)
- [ ] **Hallucination-Validator 서브 에이전트 도입**: 중요한 결정이나 코드 변경 직후, 해당 변경 사항이 기존 시스템의 문서 및 코드와 논리적 모순이 없는지 교차 검증하는 전용 서브 에이전트 개발
- [ ] **자가 개선 피드백 루프(Self-Healing Loop) 강화**: 할루시네이션이나 에러 발생 시 이를 데이터셋으로 삼아 `AGENTS.md` 및 가이드라인을 동적으로 업데이트하는 자동화 프로세스 구축
