# 📚 Documentation Roles & Boundaries

> **목적**: 각 문서 파일의 명확한 역할과 경계를 정의하여 중복과 역할 침범을 방지

## 🎯 **핵심 원칙**

1. **Single Responsibility**: 각 파일은 하나의 명확한 목적만 가짐
2. **No Duplication**: 동일한 정보는 한 곳에만 위치
3. **Clear Boundaries**: 어떤 내용이 어느 파일에 속하는지 명확한 기준
4. **Cross-Reference**: 중복 대신 참조 링크 사용

---

## 📁 **각 파일별 역할 정의**

### **📈 PROGRESS.md**
**역할**: 완료된 작업 기록 및 현재 상태 보고
**포함 내용**:
- ✅ **완료된 작업들**: 언제, 무엇을, 어떻게 완료했는지
- 🎯 **현재 프로젝트 상태**: 브랜치, 커밋, 파일 구조
- 📊 **성과 지표**: 달성한 목표, 개선 사항
- 🗂️ **파일 구조 현황**: 현재 디렉토리 및 파일 상태
- 🧠 **기술 컨텍스트**: PARA 설정, Multi-AI 환경 상태

**제외 내용**:
- ❌ 앞으로 할 일 (→ PLAN.md)
- ❌ 사용법 가이드 (→ README.md)
- ❌ 아키텍처 설명 (→ SYSTEM.md)

### **📋 PLAN.md**
**역할**: 앞으로 할 일과 우선순위 계획
**포함 내용**:
- 🚀 **Priority별 작업**: Priority 1, 2, 3 우선순위 작업
- 📅 **미완료 태스크**: [ ] 체크박스로 표시된 할 일들
- 🎯 **즉시 실행 가능한 작업**: 바로 착수할 수 있는 작업
- ⚡ **체크리스트**: 새 세션 시작 시 확인할 것들
- 💡 **다음 작업자 가이드**: 권장 사항 및 주의사항

**제외 내용**:
- ❌ 완료된 작업 (→ PROGRESS.md)
- ❌ 상세한 명령어 (→ AI_COMMON_INSTRUCTIONS.md)
- ❌ 시스템 설명 (→ SYSTEM.md)

### **🤖 AI_COMMON_INSTRUCTIONS.md**
**역할**: AI와 사용자 공통 작업 워크플로우
**포함 내용**:
- 👤 **사용자 세션 시작 가이드**: 프로젝트 복귀 시 절차
- 🤖 **AI 작업 시작 플로우**: AI가 따라야 할 워크플로우
- 📋 **필수 체크리스트**: 작업 전 반드시 확인할 것들
- 🎯 **역할 분담**: 사용자 vs AI 역할 구분
- ⚡ **빠른 명령어**: 자주 사용하는 상태 확인 명령어

**제외 내용**:
- ❌ 구체적인 Priority (→ PLAN.md)
- ❌ 완료 기록 (→ PROGRESS.md)
- ❌ 시스템 아키텍처 (→ SYSTEM.md)
- ❌ 파일별 사용법 (→ CLAUDE.md, GEMINI.md)

### **🏗️ SYSTEM.md**
**역할**: 시스템 아키텍처 및 기술 참조
**포함 내용**:
- 🏗️ **시스템 아키텍처**: 전체 구조와 모듈 관계
- ⚙️ **PARA 분류 규칙**: 기술적 구현 및 설정
- 🔧 **확장 가이드**: 새 AI 추가, 커스텀 등
- 🧪 **테스트 및 검증**: 품질 보증 방법
- 📊 **성능 최적화**: 기술적 개선 방법

**제외 내용**:
- ❌ 작업 우선순위 (→ PLAN.md)
- ❌ 완료 기록 (→ PROGRESS.md)
- ❌ 사용자 가이드 (→ README.md)

### **📖 README.md**
**역할**: 프로젝트 소개 및 사용법
**포함 내용**:
- 🎯 **프로젝트 개요**: 무엇을 하는 프로젝트인지
- ✨ **핵심 기능**: 주요 기능과 특징
- 🚀 **빠른 시작**: 설치 및 기본 사용법
- 📚 **문서 구조**: 다른 문서들로의 안내
- 💡 **Best Practices**: 효율적 사용 방법

**제외 내용**:
- ❌ 상세한 아키텍처 (→ SYSTEM.md)
- ❌ 작업 진행사항 (→ PROGRESS.md, PLAN.md)
- ❌ AI별 세부사항 (→ CLAUDE.md, GEMINI.md)

### **🔵 CLAUDE.md / 🟡 GEMINI.md**
**역할**: 특정 AI 환경 설정 및 사용법
**포함 내용**:
- 🛠️ **환경 설정**: 해당 AI CLI 설치 및 인증
- ⚙️ **설정 최적화**: 해당 AI에 특화된 설정
- 🚨 **문제 해결**: 해당 AI 관련 troubleshooting
- 🔗 **공통 시스템 연동**: 다른 문서로의 참조

**제외 내용**:
- ❌ 다른 AI와의 비교 (이미 제거됨)
- ❌ 공통 워크플로우 (→ AI_COMMON_INSTRUCTIONS.md)
- ❌ 시스템 아키텍처 (→ SYSTEM.md)

## 🚨 **경계 위반 금지 사항**

### **절대 하면 안되는 것들**
1. **PLAN.md에 완료 작업 기록하기** - PROGRESS.md로
2. **AI_COMMON_INSTRUCTIONS.md에 구체적 Priority 나열** - PLAN.md로
3. **동일한 명령어/체크리스트 여러 파일에 복사** - 한 곳에만
4. **PROGRESS.md에 미래 계획 쓰기** - PLAN.md로
5. **README.md에 작업 진행상황 쓰기** - PROGRESS.md로

### **올바른 처리 방법**
- ✅ **참조 링크 사용**: `자세한 내용은 [SYSTEM.md](SYSTEM.md) 참조`
- ✅ **요약 + 참조**: 핵심만 간단히 + 상세 내용 링크
- ✅ **역할에 맞는 배치**: 각 내용을 정확한 파일에

## 📋 **컨텐츠 배치 결정 트리**

새로운 내용을 어느 파일에 넣을지 결정할 때:

```
1. 이 내용이 "완료된 것"인가?
   YES → PROGRESS.md
   NO  → 2번으로

2. 이 내용이 "앞으로 할 일"인가?
   YES → PLAN.md
   NO  → 3번으로

3. 이 내용이 "작업 워크플로우"인가?
   YES → AI_COMMON_INSTRUCTIONS.md
   NO  → 4번으로

4. 이 내용이 "시스템 구조/기술"인가?
   YES → SYSTEM.md
   NO  → 5번으로

5. 이 내용이 "프로젝트 소개/사용법"인가?
   YES → README.md
   NO  → 6번으로

6. 이 내용이 "특정 AI 환경 설정"인가?
   YES → CLAUDE.md 또는 GEMINI.md
   NO  → 어디에도 안맞으면 가장 가까운 파일에 + 이 문서 업데이트
```
