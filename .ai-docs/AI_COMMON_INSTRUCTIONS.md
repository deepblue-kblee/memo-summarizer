# 🤖 AI 공통 작업 지침

> **모든 AI (Claude, Gemini, 기타)와 사용자가 작업 시작 전 반드시 읽어야 하는 공통 지침**

## 🚨 **절대 원칙 (대전제) - MUST READ FIRST**

### **⚠️ 모든 AI가 반드시 준수해야 하는 4가지 대전제**

#### **1. 🎯 AI 중립성 원칙**
**AI가 무엇이든 상관없다. AI 선택은 사용자가 한다.**
- ✅ Claude든 Gemini든 OpenAI든 **동일한 개발 작업** 수행
- ✅ AI별 특성 비교하거나 "어떤 AI가 더 좋다" 판단 금지
- ✅ 사용자가 선택한 AI로 **목적 달성에만 집중**

#### **2. 🚀 목적 집중 원칙**
**AI는 이 repo의 개발 목적에만 집중한다. 비교/실험/테스트 목적이 아니다.**
- ✅ 메모 분석, 코드 수정, 문서 업데이트 등 **repo 개발 작업**에만 집중
- ❌ AI 성능 테스트, AI 간 비교, AI 능력 실험 금지
- ❌ "Claude가 더 정확해", "Gemini가 더 빨라" 같은 평가 금지

#### **3. 🔄 작업 연속성 원칙**
**어떤 AI든 이전 작업을 이어받아서 즉시 개발할 수 있어야 한다.**
- ✅ PROGRESS.md → PLAN.md 읽고 **현재 상태 + 다음 할 일** 파악
- ✅ 다른 AI가 했던 작업을 **seamless하게 이어받기**
- ✅ 작업 완료시 다음 AI를 위해 **상태 업데이트**

#### **4. 📋 투명성 원칙**
**사용자가 repo 상태와 다음 할 일을 명확히 파악할 수 있어야 한다.**
- ✅ 모든 작업을 PROGRESS.md에 **명확히 기록**
- ✅ 다음 우선순위를 PLAN.md에 **구체적으로 명시**
- ✅ 사용자가 언제든 **현재 상황을 5분 내 파악** 가능하도록

### **💡 대전제 요약**
```
AI 선택 = 사용자 몫
AI 역할 = repo 개발 목적 달성
AI 협업 = 환경에 상관없이 작업 연속성 보장
사용자 = 언제든 명확한 상황 파악 가능
```

## 👤 **사용자(사람) 세션 시작 가이드**

### **📋 사용자 빠른 시작 체크리스트**

새로운 세션을 시작하거나 프로젝트에 돌아왔을 때:

#### **1단계: 프로젝트 현황 파악 (2분)**
```bash
# 최근 변경사항 확인
git log --oneline -5
git status

# 현재 진행 상황 빠른 확인
head -30 .ai-docs/PROGRESS.md  # 최근 완료된 작업들
head -20 .ai-docs/PLAN.md      # 다음 우선순위 작업들
```

#### **2단계: 사용자 읽기 순서**
**🚀 빠른 복귀 (5분)**: 이미 프로젝트를 알고 있는 경우
```bash
1. PROGRESS.md (2분) → 어디까지 완료되었는지
2. PLAN.md (2분) → 다음에 무엇을 해야 하는지
3. 바로 작업 시작 또는 AI에게 작업 지시
```

**📚 전체 이해 (15분)**: 처음 참여하거나 전체 맥락이 필요한 경우
```bash
1. README.md (5분) → 프로젝트 전체 개요
2. SYSTEM.md (7분) → 시스템 아키텍처 이해
3. PROGRESS.md (2분) → 현재 완료 상태
4. PLAN.md (1분) → 다음 작업 확인
```

#### **3단계: 작업 방향 결정**

**🤖 AI에게 맡길 작업**
```bash
✅ "현재 상황 요약해줘"           → 사용 가능한 AI로 수행
✅ "Priority 1 작업 진행해줘"     → PLAN.md 확인 후 진행
✅ "문서 업데이트해줘"           → 사용 가능한 AI로 수행
✅ "코드 구현해줘"               → 사용 가능한 AI로 수행
✅ "파일 처리해줘"              → 사용 가능한 AI로 수행
```

**👤 직접 할 작업**
```bash
✅ 프로젝트 방향 결정             → 사용자 판단 필요
✅ 비즈니스 요구사항 정의         → 도메인 지식 필요
✅ 외부 도구 설치/설정           → 환경 의존적
✅ Git 브랜치 전략 결정          → 팀 정책에 따라
✅ AI 작업 결과 검토 및 승인      → 최종 품질 관리
```

### **⚡ 상황별 빠른 가이드**

#### **🔴 긴급 상황**: "지금 당장 뭘 해야 하지?"


### **🎯 핵심 원칙: 필요시에만, 사용자 승인 후**

### **📊 스마트 트리거 시스템**

#### **Step 1: 세션 시작 시 간단 체크 (5초, <10 토큰)**
```bash
# .ai-docs 변경 누적도 확인
changes=$(git log --oneline --since="$(cat .ai-docs/.last_cleanup 2>/dev/null || echo '1 week ago')" .ai-docs/ | wc -l)
if [ $changes -gt 10 ]; then
    echo "📋 문서 정리 검토 권장 (${changes}회 변경 누적)"
fi
```

#### **Step 2: 조건부 사용자 질문**
**트리거 조건**:

- ✅ **개별 파일 5회 이상** 수정 시

> "📋 문서가 많이 수정되었습니다 (누적: N회). 정리를 진행할까요?"
