# 🤖 AI 공통 작업 지침

> **모든 AI (Claude, Gemini, 기타)와 사용자가 작업 시작 전 반드시 읽어야 하는 공통 지침**

## 👤 **사용자(사람) 세션 시작 가이드**

### **📋 사용자 빠른 시작 체크리스트**

새로운 세션을 시작하거나 프로젝트에 돌아왔을 때:

#### **1단계: 프로젝트 현황 파악 (2분)**
```bash
# 최근 변경사항 확인
git log --oneline -5
git status

# 현재 진행 상황 빠른 확인
head -30 .ai-docs/PROGRESS.md  # 최근 완료된 작업들
head -20 .ai-docs/PLAN.md      # 다음 우선순위 작업들
```

#### **2단계: 사용자 읽기 순서**
**🚀 빠른 복귀 (5분)**: 이미 프로젝트를 알고 있는 경우
```bash
1. PROGRESS.md (2분) → 어디까지 완료되었는지
2. PLAN.md (2분) → 다음에 무엇을 해야 하는지
3. 바로 작업 시작 또는 AI에게 작업 지시
```

**📚 전체 이해 (15분)**: 처음 참여하거나 전체 맥락이 필요한 경우
```bash
1. README.md (5분) → 프로젝트 전체 개요
2. SYSTEM.md (7분) → 시스템 아키텍처 이해
3. PROGRESS.md (2분) → 현재 완료 상태
4. PLAN.md (1분) → 다음 작업 확인
```

#### **3단계: 작업 방향 결정**

**🤖 AI에게 맡길 작업**
```bash
✅ "현재 상황 요약해줘"           → Claude/Gemini 둘 다 가능
✅ "Priority 1 작업 진행해줘"     → PLAN.md 확인 후 적절한 AI 선택
✅ "문서 업데이트해줘"           → Claude (정확성 중요)
✅ "코드 구현해줘"               → Claude (복잡한 분석)
✅ "대량 파일 처리해줘"          → Gemini (속도 우선)
```

**👤 직접 할 작업**
```bash
✅ 프로젝트 방향 결정             → 사용자 판단 필요
✅ 비즈니스 요구사항 정의         → 도메인 지식 필요
✅ 외부 도구 설치/설정           → 환경 의존적
✅ Git 브랜치 전략 결정          → 팀 정책에 따라
✅ AI 작업 결과 검토 및 승인      → 최종 품질 관리
```

### **⚡ 상황별 빠른 가이드**

#### **🔴 긴급 상황**: "지금 당장 뭘 해야 하지?"
```bash
1. grep -A 5 "🚨" .ai-docs/PLAN.md     # 긴급 작업 확인
2. git status                          # 현재 변경사항 확인
3. AI에게 "긴급 상황 파악 및 대응" 요청
```

#### **🟡 정기 점검**: "일주일 만에 돌아왔는데..."
```bash
1. git log --oneline --since="1 week ago"  # 최근 1주일 변경사항
2. tail -20 .ai-docs/PROGRESS.md           # 최근 완료 작업
3. head -10 .ai-docs/PLAN.md               # 현재 우선순위
```

#### **🟢 새로 참여**: "이 프로젝트가 뭐지?"
```bash
1. cat README.md                           # 프로젝트 소개
2. head -50 .ai-docs/SYSTEM.md            # 시스템 개요
3. AI에게 "프로젝트 전체 설명해줘" 요청
```
